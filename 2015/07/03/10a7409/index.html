kafka Expanding your cluster -

By edwin

发表于 2015-07-03

文章目录

前面我们讲kafka的安装，添加一个broker是非常容易的，只需要确保broker_id是唯一的，并成功启动它，便已经差不点添加成功了，但是这些新服务器不会自动被分配任何数据分区,所以除非将分区移动过来，否则他们不会做任何工作，直到新创建的topics。所以通常当你添加机器集群需要一些现有的数据迁移到这些机器上来。

迁移前，我们先producer生产一些消息，consumer消费记录，我们将其记录下来：

Shell

[root@node3 ~]# /usr/local/kafka/bin/kafka-topics.sh --zookeeper node1:2181,node2:2181,node3:2181 --topic my-reassign-test --describe
Topic:my-reassign-test  PartitionCount:4    ReplicationFactor:2     Configs:
    Topic: my-reassign-test Partition: 0    Leader: 2       Replicas: 2,1   Isr: 2,1
    Topic: my-reassign-test Partition: 1    Leader: 0       Replicas: 0,2   Isr: 0,2
    Topic: my-reassign-test Partition: 2    Leader: 1       Replicas: 1,0   Isr: 1,0
    Topic: my-reassign-test Partition: 3    Leader: 2       Replicas: 2,0   Isr: 2,0
[root@node3 ~]# /usr/local/kafka/bin/kafka-console-consumer.sh --zookeeper node1:2181,node2:2181,node3:2181 --topic my-reassign-test --from-beginning

partition reassignment tool可以运行三个互斥的模式，下面通过实例看看是怎么一回事：

1、先创建一个json文件：

Shell

# cat topics-to-move.json 

{&quot;topics&quot;: [{&quot;topic&quot;: &quot;my-reassign-test&quot;}],
 &quot;version&quot;:1
}

2、执行–generate：

Shell

# /usr/local/kafka/bin/kafka-reassign-partitions.sh --zookeeper node1:2181,node2:2181,node3:2181 --topics-to-move-json-file /root/topics-to-move.json --broker-list &quot;3,4&quot; --generate

Current partition replica assignment

{&quot;version&quot;:1,&quot;partitions&quot;:[{&quot;topic&quot;:&quot;my-reassign-test&quot;,&quot;partition&quot;:0,&quot;replicas&quot;:[2,1]},{&quot;topic&quot;:&quot;my-reassign-test&quot;,&quot;partition&quot;:3,&quot;replicas&quot;:[2,0]},{&quot;topic&quot;:&quot;my-reassign-test&quot;,&quot;partition&quot;:1,&quot;replicas&quot;:[0,2]},{&quot;topic&quot;:&quot;my-reassign-test&quot;,&quot;partition&quot;:2,&quot;replicas&quot;:[1,0]}]}
Proposed partition reassignment configuration

{&quot;version&quot;:1,&quot;partitions&quot;:[{&quot;topic&quot;:&quot;my-reassign-test&quot;,&quot;partition&quot;:0,&quot;replicas&quot;:[3,4]},{&quot;topic&quot;:&quot;my-reassign-test&quot;,&quot;partition&quot;:3,&quot;replicas&quot;:[4,3]},{&quot;topic&quot;:&quot;my-reassign-test&quot;,&quot;partition&quot;:1,&quot;replicas&quot;:[4,3]},{&quot;topic&quot;:&quot;my-reassign-test&quot;,&quot;partition&quot;:2,&quot;replicas&quot;:[3,4]}]}

输出两部分，前部分作为备份，以便回滚，后半部署保存为expand-cluster-reassignment.json以便下面-execute使用,执行操作还并没有开始

3、执行 –execute：

Shell

# /usr/local/kafka/bin/kafka-reassign-partitions.sh --zookeeper node1:2181,node2:2181,node3:2181 --reassignment-json-file /root/expand-cluster-reassignment.json --execute
Current partition replica assignment

{&quot;version&quot;:1,&quot;partitions&quot;:[{&quot;topic&quot;:&quot;my-reassign-test&quot;,&quot;partition&quot;:0,&quot;replicas&quot;:[2,1]},{&quot;topic&quot;:&quot;my-reassign-test&quot;,&quot;partition&quot;:3,&quot;replicas&quot;:[2,0]},{&quot;topic&quot;:&quot;my-reassign-test&quot;,&quot;partition&quot;:1,&quot;replicas&quot;:[0,2]},{&quot;topic&quot;:&quot;my-reassign-test&quot;,&quot;partition&quot;:2,&quot;replicas&quot;:[1,0]}]}

Save this to use as the --reassignment-json-file option during rollback
Successfully started reassignment of partitions {&quot;version&quot;:1,&quot;partitions&quot;:[{&quot;topic&quot;:&quot;my-reassign-test&quot;,&quot;partition&quot;:0,&quot;replicas&quot;:[3,4]},{&quot;topic&quot;:&quot;my-reassign-test&quot;,&quot;partition&quot;:3,&quot;replicas&quot;:[4,3]},{&quot;topic&quot;:&quot;my-reassign-test&quot;,&quot;partition&quot;:1,&quot;replicas&quot;:[4,3]},{&quot;topic&quot;:&quot;my-reassign-test&quot;,&quot;partition&quot;:2,&quot;replicas&quot;:[3,4]}]}

正常执行的话会生成同上图类似的json串，表示原始状态和目标状态

4、–verify模式：

Shell

# /usr/local/kafka/bin/kafka-reassign-partitions.sh --zookeeper node1:2181,node2:2181,node3:2181 --reassignment-json-file /root/expand-cluster-reassignment.json --verify
Status of partition reassignment:
Reassignment of partition [my-reassign-test,0] completed successfully
Reassignment of partition [my-reassign-test,3] completed successfully
Reassignment of partition [my-reassign-test,1] completed successfully
Reassignment of partition [my-reassign-test,2] completed successfully
